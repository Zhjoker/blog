# 思考 🤔

  在完成前端朋友的这个任务后，我开始思考，除了if else,还有哪几种判断方式呢。

# 结论

  在一番苦思冥想🧘‍♂️后我终于得到了四个答案，并总结出了他们的特点。

# 一.三元表达式(适合比较简单的判断)


// 判断price 是否为空 不存在的话以 --- 代替
// * 三元表达式
price= price === null ? price : '---'
// * if else 判断
if(price=== null){
  price='---';
}else{
  price=price;
}

# 二. || (或 适合判断简单的)

|| 你没有看错就是或 。比如，我们要做分页，如果没有下一页就要告诉用户到最后一页了，到底了。数据中也没有到最后一页的提示 用 或 怎么写呢。

let pageSize = 9; // 一个数据量
let isHasMore = false; // 是否有更多数据
http
       .get("url", {
         currPage: page,
         pageSize:pageSize,
       })
       .then(res => {
          //  ...
          let dataList=res.list //数据列表
          // 数据的长度 比 一个获取的数量小 说明更多没有数据 
          dataList.lenght<this.pageSize || (this.isHasMore=false);
     
       });

# 三.switch(适合较为复杂的判断)

这就是文章开始用到的一整个判断方式，下面拿一个更加好的例子🌰来说。

/**
 * 按钮点击事件
 * @param {number} status 活动状态：1 开团进行中 2 开团失败 3 商品售罄 4 开团成功 5 系统取消
 */
const statusChange = (status)=>{
  if(status == 1){
    sendLog('processing')
    jumpTo('IndexPage')
  }else if(status == 2 && status == 3){
    sendLog('fail')
    jumpTo('FailPage')
  }else if(status == 4){
    sendLog('success')
    jumpTo('SuccessPage')
  }else if(status == 5){
    sendLog('cancel')
    jumpTo('CancelPage')
  }else {
    sendLog('other')
    jumpTo('Index')
  }
}

/**
 * 按钮点击事件
 * @param {number} status 活动状态：1 开团进行中 2 开团失败 3 商品售罄 4 开团成功 5 系统取消
 */
const onButtonClick = (status)=>{
  switch (status){
    case 1:
      sendLog('processing')
      jumpTo('IndexPage')
      break
    case 2:
    case 3:
      sendLog('fail')
      jumpTo('FailPage')
      break  
    case 4:
      sendLog('success')
      jumpTo('SuccessPage')
      break
    case 5:
      sendLog('cancel')
      jumpTo('CancelPage')
      break
    default:
      sendLog('other')
      jumpTo('Index')
      break
  }
}

# 四.策略模式（适合较复杂的判断）

还是上面的那段代码为例。同学们看到下面这个例子，这个例子用的策略模式，把每一种可能发生的，对应做出一种不同的策略应对，看起来清晰，又简洁。 同时也可以清楚的告诉每一个看代码的人，所以存在的情况，让人更容易理解。

还有很多地方也可以用到这种策略判断，如动态表单，每添加一个表单的选项就加入一种对应的策略，等等。
